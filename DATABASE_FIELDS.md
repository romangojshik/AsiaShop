# Документация полей базы данных

## Коллекция: `sushi` (Суши)

Описание полей для документов в коллекции суши.

| Поле | Тип | Обязательное | Описание | Пример |
|------|-----|--------------|----------|--------|
| `id` | String | ✅ Да | Уникальный идентификатор продукта | `"1"` |
| `title` | String | ✅ Да | Название суши | `"Асами"` |
| `imageURL` | String | ✅ Да | Имя изображения (без расширения) | `"asami"` |
| `price` | Number | ✅ Да | Цена в рублях | `65` |
| `description` | String | ✅ Да | Подробное описание продукта | `"Сочный лосось, кремовый сыр..."` |
| `weight` | String | ❌ Нет | Вес продукта | `"930г"` |
| `callories` | String | ❌ Нет | Калорийность | `"1200ккал"` |
| `composition` | String | ❌ Нет | Состав продукта | `"Лосось, сыр, огурец..."` |

---

## Коллекция: `sets` (Сеты)

Описание полей для документов в коллекции сетов.

| Поле | Тип | Обязательное | Описание | Пример |
|------|-----|--------------|----------|--------|
| `id` | String | ✅ Да | Уникальный идентификатор сета | `"1"` |
| `title` | String | ✅ Да | Название сета | `"Ямонато"` |
| `imageURL` | String | ✅ Да | Имя изображения (без расширения) | `"yamonato"` |
| `price` | Number | ✅ Да | Цена в рублях | `80` |
| `description` | String | ✅ Да | Подробное описание сета | `"Разнообразный набор роллов..."` |
| `weight` | String | ❌ Нет | Вес сета | `"1400г"` |
| `callories` | String | ❌ Нет | Калорийность | `"2200ккал"` |
| `composition` | String | ❌ Нет | Состав сета | `"Лосось, угорь, креветка..."` |

---

## Пример документа

```json
{
  "id": "1",
  "title": "Асами",
  "imageURL": "asami",
  "price": 65,
  "description": "Сочный лосось, кремовый сыр, хрустящие огурец и снежный краб. Идеальный баланс в каждом кусочке. Попробуй яркое настроение!",
  "weight": "930г",
  "callories": "1200ккал",
  "composition": "Лосось, сыр, огурец, снежный краб, тобико."
}
```

---

## Примечания

- **Обязательные поля** должны присутствовать во всех документах
- **Необязательные поля** могут отсутствовать (будут `nil` в Swift)
- `imageURL` хранится без расширения файла (`.png`, `.jpg` и т.д.)
- `price` хранится как число (Number), не строка
- `weight` и `callories` хранятся как строки с единицами измерения
